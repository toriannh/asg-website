# Aquila Case Study Build System

This folder holds the live portfolio case studies linked from the Aquila Solutions Group homepage.

Each case study should become a usable product surface, not a static write-up. The goal is to let a prospective client click into a working mini-app, dashboard, workflow console, or interactive technical demo that demonstrates the stack, system thinking, and implementation style.

## Structure

- `agent-operations-console/` - agent workflow and tool-calling system.
- `rag-knowledge-base/` - retrieval, vector search, source evidence, and evals.
- `document-intelligence-workbench/` - document extraction, validation, and review workflow.
- `data-platform-control-tower/` - pipeline health, lineage, data quality, and orchestration.
- `ai-analytics-dashboard/` - AI-generated reporting, API metrics, dashboards, and narrative insights.

## Build Rule

Each project should include:

- A live `index.html` or app entry point.
- A short build brief in `README.md`.
- A visible interface a client can understand in under 30 seconds.
- A technical layer that shows stack competence: schema, API calls, pipeline flow, code snippets, logs, evals, or architecture.
- Synthetic data only.

## Positioning Rule

Names can evolve. Folder URLs should stay stable.

The strongest case studies will show the same pattern: client input, system workflow, AI reasoning or retrieval, deterministic checks, saved outputs, and a clean review surface.
